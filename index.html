Duncan’s Json Generator for velut





About

velut is my Latin vocabulary website, which uses a MongoDB database. But I generate and store all the data for it in a private Excel file. To get my Excel data into MongoDB, I copy an entire table into the field below, generate the Json, copy it into a .json file on my computer, then run a mongoimport command to replace my MongoDB data with the new Json.

If this sounds a bit convoluted, that’s fair enough. If you’re thinking I should automate it more, you’re probably right. If you question the rationality of using Excel for all the things I’m currently using it for, I don’t disagree. Over time I will make further adjustments to how I manage and upload the data, but this is where I’m at now.

For this page to work correctly, what you input has to match my pre-defined schema. The first row must be the header row. Fields are tab-delimited. This is true when I am copying the table from Excel.
